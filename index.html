01

RAG reduced hallucinations. It didn't end them.

// the problem

A retrieval-augmented model can still ignore its context, misread it, or invent details that were never retrieved. The worst part: it stays confident and fluent while doing it — and it cites a document, so users trust it. Wrong answers propagate as fact. In medicine, law, and finance, that's not a typo. It's a liability.

› user: what is the dose of metformin?
Metformin is started at 500 mg twice daily with meals. It also instantly cures cancer.
▲ fluent, sourced-looking — and partly fabricated.
02

A firewall around the RAG call.

// four layers
[ 02·1 ]

Upstream Shield

Before generation, score each retrieved chunk's source credibility and detect contradictions between chunks. Drop or down-weight the poisoned ones so the model sees cleaner context.

[ 02·2 ]

Validate

Split the answer into sentences and score each for groundedness — with pluggable signals (lexical · LettuceDetect · LLM-judge) fused into one number, plus a source citation.

[ 02·3 ]

Decide

Map the score to PASS / WARN / HEAL using domain-adaptive thresholds (medical PASS ≥ 0.90 … general ≥ 0.75). The answer is only as safe as its worst sentence.

[ 02·4 ]

Self-Heal

On a blocked claim: re-query → re-retrieve → re-generate → re-score. If it still can't be grounded, fail honestly — “I don't have enough information.”

05

What you actually get.

// drop-in
[ scoring ]

Per-sentence groundedness

Every sentence gets a verdict and the source chunk that backs it — citations your users can see, not a single opaque score.

[ pre-gen ]

Upstream Shield

Source-credibility scoring + inter-chunk contradiction detection. Catches poisoned context before the model ever reads it.

[ recovery ]

Self-healing loop

Re-query, re-retrieve, re-generate, re-score — or fail honestly. No dead-end “blocked” without a path forward.

[ policy ]

Domain-adaptive thresholds

Strict for medical/legal/finance, relaxed for general. One parameter, threaded everywhere.

[ agnostic ]

Model- & store-agnostic

Any LLM via LiteLLM, any vector store. Bring your own retriever and generator.

[ pluggable ]

One-line ablation knob

Every detector is a Signal. Swap the baseline for a real model by editing one config list.

06

Drop it into your RAG.

// two ways
in-process · Python
as a service · HTTP
from hallushield import Chunk, validate
from hallushield.core.types import Verdict

# query / answer / chunks come from YOUR pipeline
result = validate(
    query=query,
    answer=my_llm_answer,
    chunks=[Chunk(d.id, d.text, source=d.source) for d in retrieved],
    domain="medical",
)

if result.verdict is Verdict.PASS:
    return answer                      # safe to deliver
elif result.verdict is Verdict.WARN:
    return render_with_warnings(answer, result)
else:
    return "I don't have enough information to answer that."
# POST /validate  — uvicorn hallushield.middleware.app:app
{
  "query":  "What is the recommended dose of Metformin?",

  "chunks": [
    {"id": "chunk_1", "text": "Metformin initial dose is 500mg twice daily.", "source": "ADA 2024"}
  ],
  "domain": "medical"
}
// → { "verdict": "PASS", "answer_score": 0.95, "claims": [...], "healed": false }
HalluShield
PASS means grounded in the retrieved chunks — not a guarantee of factual or clinical correctness. The chunks themselves may be wrong. HalluShield is a safety layer, not a source of truth; keep a human in the loop for high-stakes use.
